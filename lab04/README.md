# Лабораторна робота №4: Особистий цифровий підпис (RSA)

## Опис

У цій папці розміщено вихідний код десктопного GUI-застосунку для створення та перевірки цифрових підписів. Програма реалізує асиметричне шифрування (алгоритм RSA) "з нуля", включаючи генерацію ключів, математичне обчислення підпису та його верифікацію.

Архітектура проєкту — монолітна (`main.py`) з використанням об'єктно-орієнтованого підходу (розділення на логіку RSA та інтерфейс), що забезпечує зручність запуску.

## Технічне завдання та Функціонал

Скрипт `main.py` реалізує повний цикл роботи з ЕЦП (Електронним цифровим підписом):

* **Нативна математика RSA:** Власна реалізація алгоритму без використання криптографічних бібліотек. Реалізовано тест Міллера-Рабіна для пошуку простих чисел та розширений алгоритм Евкліда.
* **Персоналізація ключів:** Генерація пари ключів (Public/Private) відбувається на основі хешу від ПІБ та дати народження користувача (використовується як seed).
* **Створення підпису:**
    * Обчислення хешу файлу (алгоритм **SHA-256**).
    * Шифрування хешу приватним ключем ($S = H^d \pmod n$).
* **Верифікація:** Розшифрування підпису публічним ключем та порівняння з реальним хешем файлу для підтвердження цілісності.
* **Захист від підробок:** Програма автоматично виявляє будь-які модифікації файлу після підписання.

## Технології

* **Мова:** Python 3.10+
* **Інтерфейс:** Tkinter (Стандартна графічна бібліотека)
* **Криптографія:** Hashlib (SHA-256), Random (Генерація великих чисел)


## Як користуватися

У графічному інтерфейсі доступні три кроки:

1.  **Генерація ключів:**
    * Введіть ПІБ та дату народження.
    * Натисніть "Згенерувати пару ключів".
    * *Результат:* Створяться файли `my_private.key` та `my_public.key`.

2.  **Підписати файл:**
    * Оберіть файл документа (наприклад, `contract.txt`).
    * Натисніть "Підписати файл".
    * *Результат:* Поруч з файлом з'явиться підпис `.sig`.

3.  **Перевірити підпис:**
    * Оберіть оригінальний файл.
    * Оберіть файл підпису `.sig`.
    * Натисніть "Перевірити".
    * *Результат:* Програма повідомить, чи дійсний підпис (зелене повідомлення) або файл було змінено (червоне попередження).
